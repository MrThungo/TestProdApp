# Back to God AOG App

Fresh Flask foundation for Back to God AOG, Gqeberha City Centre.

## Structure

```text
app.py
back_to_god/
  models.py
  extensions.py
  core/
  modules/
    auth/

    notifications/
    profile/
    public/
    search/
    timeline/
    users/
    visitors/
  services/
  static/
  templates/
instance/
requirements.txt
run_app.bat
```

## Run

Double-click:

```text
run_app.bat
```

The script prints the local address. It uses `5000`, or `5001` if `5000` is busy.

## Publish

This is a Flask backend app, so it must be published to a Python host rather than a static website host.

Production files included:

- `Procfile` for hosts that support web process commands
- `runtime.txt` for Python version selection
- `render.yaml` for Render-style deployment
- `Dockerfile` for container hosting
- `.env.example` for required production settings
- `DEPLOYMENT.md` with publishing steps

Do not publish with quick login enabled. In production, set:

```text
ENABLE_QUICK_LOGIN=0
SESSION_COOKIE_SECURE=1
SECRET_KEY=<long-random-secret>
```

## First Login

The first super admin password is generated here:

```text
instance\initial-super-admin.txt
```

Passwords for new users are generated by the app and stored as hashes.

## Storage Notes

- The app uses SQLite at `instance\back_to_god.sqlite3`.
- SQLAlchemy models live in `back_to_god\models.py`; the existing services still keep the app fast and stable while the ORM layer is available for the next phase.
- Passwords are stored as hashes.
- Password reset links are stored as hashed 5-minute tokens.
- Account and reset emails use SMTP when configured; otherwise they are written to `instance\email-outbox.log`.
- Profile photos are stored as files, not inside the database.
- Live video is compressed in the browser, uploaded once when the live ends, and stored as one database recording.
- SQLite uses WAL mode and a longer busy timeout to reduce database-lock errors during media uploads and reads.
- Message attachments and timeline media are saved as database bytes.
- Videographer video uploads are not blocked by the app-level video size cap.
- Message and notification text is compacted before saving.

## External Notification API

The app includes a free API endpoint for outside tools to create in-app notifications:

```text
POST /api/notifications/send
```

The API key is generated on startup in:

```text
instance\external-notification-api-key.txt
```

Send it as `X-API-Key` or `Authorization: Bearer <key>`.

Example JSON:

```json
{
  "title": "Choir practice",
  "message": "Choir practice starts at 18:00 tonight.",
  "target_url": "/announcements/",
  "role": "member"
}
```

Use `user_ids: [1, 2, 3]` to notify specific people. If no `role` or `user_ids` is sent, every active user receives the notification.

## Frontend

- Tailwind CSS is loaded through the browser CDN for this early Flask version.
- Material Symbols and local Material-style component classes are used for icons, buttons, cards, fields, and mobile navigation.
- Light and dark mode can be switched from the header. The preference is saved in the `btg_theme` cookie.
- Full MUI is React-only, so it is not installed in this Flask/Jinja version. If the frontend later moves to React, these screens can be mapped to MUI components cleanly.
